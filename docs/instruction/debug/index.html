Отладка JavaScript в различных средах разработки

В этом руководстве мы рассмотрим, как запустить код на JavaScript в режиме отладки и найти в нем ошибки, используя три различных инструмента: Visual Studio Code (VSCode), Microsoft Visual Studio и консоль браузера.

Пример кода​

Для начала давайте используем следующий код, который мы собираемся отлаживать:

function isPrime(num) {
    if (num < 2) return false;
    for (let i = 0; i <= Math.sqrt(num); i++) { // Ошибка: цикл начинается с 0
        if (num % i === 0) return false; // Ошибка: деление на 0
    }
    return true;
}

function printPrimes(arr, n) {
    let count = 0;
    for (let num of arr) {
        if (isPrime(num)) {
            console.log(num);
            count++;
            if (count === n) break;
        }
    }
    if (count < n) {
        console.log("Не найдено достаточно простых чисел.");
    }
}

const arr = [1, 2, 3, 6, 8, 11, 13, 20];
let n = 3; // Вы можете задать любое значение для n
printPrimes(arr, n);

Запуск отладки в Visual Studio Code​

  1. Настройка отладчика:

    • Убедитесь, что у вас установлен Node.js. Это необходимо для выполнения JavaScript кода вне браузера.
    • Откройте VSCode и создайте новый файл, например, index.js. Вставьте туда наш код.

  2. Создание конфигурации отладки:

    • Перейдите на панель отладки, нажав Ctrl + Shift + D.
    • Нажмите на иконку шестеренки (⚙️) для создания конфигурации отладки.
    • Выберите Node.js из предложенных вариантов.

  3. Запуск отладки:

    • Установите точки останова, щелкнув слева от номера строки, где хотите остановить выполнение.
    • Нажмите F5 или выберите Run > Start Debugging из меню. Код запустится в режиме отладки, и выполнение остановится на установленных точках останова.

  4. Проверка значений переменных:

    • Используйте панель "Variables", чтобы проверять значения переменных во время выполнения кода. Обратите внимание на значения переменных i и num в функции isPrime, чтобы выявить, почему код может возвращать неправильные результаты.

  5. Поиск и сиправление ошибок

    • Обратите внимание, что цикл в функции isPrime начинается с 0, что вызывает ошибку деления на ноль. Исправьте начальное значение i на 2, чтобы избежать этой проблемы.

Запуск отладки в Microsoft Visual Studio​

  1. Создание нового проекта:

    • Откройте Microsoft Visual Studio и создайте новый проект Node.js (если у вас установлены необходимые расширения).
    • Вставьте наш код в созданный файл.

  2. Установка точек останова:

    • Установите точки останова, щелкнув по левому краю номера строки кода.

  3. Запуск отладки:

    • Нажмите F5 для запуска проекта в режиме отладки. Visual Studio остановит выполнение на установленных точках останова.

  4. Проверка значений переменных:

    • Используйте панель "Variables", чтобы проверять значения переменных во время выполнения кода. Обратите внимание на значения переменных i и num в функции isPrime, чтобы выявить, почему код может возвращать неправильные результаты.

  5. Поиск и исправление ошибок:

    • Обратите внимание, что цикл в функции isPrime начинается с 0, что вызывает ошибку деления на ноль. Исправьте начальное значение i на 2, чтобы избежать этой проблемы. Это позволит правильно проверять, является ли число простым, и предотвратит ошибку в коде.

Заключение​

Теперь вы знаете, как отлаживать код на JavaScript в различных средах разработки. Используйте эти методы, чтобы находить и исправлять ошибки в вашем коде. Отладка - важная часть разработки, и навыки, полученные в процессе, помогут вам создавать более качественные и надежные программы.
